Refactoring Legacy Applications

What is Legacy Code?

  • Code you don't like
  • Old code
  • Code that is difficult to understand/change
  • Code without tests
Source: "Working Effectively with Legacy Code" by Michael C. Feathers

What are We Trying to Do Here?

  • Modify functionality?
  • Trivial transformations?
  • Restructure internals for testability

Avoiding Regressions

Bugs will arise as unintended consequences of our refactoring.

If they're unintended, how can we prepare for them?

By using integration tests!

The Importance of Version Control

Lets augment the TDD workflow:

  1. Write a test
  2. Watch it fail
  3. Write the bare minimum to make it pass
  4. Check in changes
  5. Re-factor as necessary
  6. Check in changes
